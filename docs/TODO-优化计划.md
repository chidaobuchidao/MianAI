# 简历优化模块 — 待优化计划

> 高优项（#1~#4）已于 2026-05-13 全部完成。新增 2 项待开发功能。

## 高优先级 ✅ 已完成

| # | 问题 | 状态 |
|---|------|------|
| 1 | 小程序硬编码 LAN IP | ✅ 统一到 `request.ts` 的 `BASE_URL` |
| 2 | 深度优化失败后无重试机制 | ✅ SSE 流式 + 重试面板（最多 3 次） |
| 3 | 分析超时后无中间状态保存 | ✅ 每 5s 保存 partialResponse，支持断点续传 |
| 4 | 上传页无已有简历的"继续优化"入口 | ✅ 简历历史列表 + 状态标签 + 左滑删除 |

## 高优先级 🚧 待开发

| # | 问题 | 方案 |
|---|------|------|
| 16 | 简历预览功能未完成 | 服务器端将优化后简历转为 HTML/图片，小程序内 web-view 渲染预览 |

## 中优先级

| # | 问题 | 方案 |
|---|------|------|
| 5 | 模板生成需 AI 调用，首次可能 30s+ | 预生成常用岗位模板缓存，或使用更轻量的模型 |
| 6 | Word 预览实际是下载+系统打开，非真内嵌 | 服务器端 .docx→图片或 PDF 转换后 web-view 渲染 |
| 7 | 深度优化计时器在页面切换后重置 | 基于服务器时间戳计算已用时间 |
| 8 | AI 评分偶尔 JSON 格式错误导致解析失败 | 添加 JSON 修复逻辑（补全括号、去除多余换行） |
| 9 | 用户多次上传同名文件无法区分 | 添加文件 hash 去重，避免重复解析 |

| # | 问题 | 方案 |
